## 概述

一个结合末世生存游戏和AI实时小说生成的互动体验系统。玩家在游戏中自由行动，AI根据玩家行为和随机事件推进世界状态，同时在后台生成文学化的小说章节。

### 核心特性

- 3个预设末世场景（全球冰封、丧尸末世、核战废墟）
- 玩家可自定义创建场景
- 15+属性RPG系统
- 选项引导 + 自由输入交互
- 关键事件触发小说章节生成
- 游戏数据文学化转化为小说内容
- 短篇小说（5,000-10,000字），未来扩展中长篇

## 玩家属性系统

### 属性分类

**生存属性 (0-100)**
- 生命值 HP - 归零即死亡
- 饥饿度 - 持续下降，归零扣HP
- 口渴度 - 持续下降，归零扣HP
- 体温 - 冰封场景关键，过高/过低扣HP
- 精神值 - 归零触发精神崩溃事件

**能力属性 (0-100)**
- 力量 - 影响战斗、搬运
- 敏捷 - 影响逃跑、潜行
- 智力 - 影响解谜、制作
- 魅力 - 影响交易、说服NPC
- 感知 - 影响搜索、发现危险

**状态标签**
- 疾病状态：健康/感冒/发烧/感染/...
- 装备耐久：武器/护甲当前状态
- 特殊状态：受伤/中毒/辐射/变异...

**背包系统**
- 物品栏：食物、水、武器、材料等
- 容量限制：基于力量属性

## 场景种子系统

### 预设场景

| 场景 | 核心威胁 | 叙事风格 |
|------|---------|---------|
| 全球冰封 | 寒冷、饥饿 | 冷酷绝望、人性考验 |
| 丧尸末世 | 感染、尸潮 | 紧张恐怖、生存本能 |
| 核战废墟 | 辐射、变异 | 苍凉悲壮、文明反思 |

### 场景创建流程

```
开始游戏 → 选择场景 → 预设场景直接进入 / 自定义场景
                     ↓
              场景微调面板 / 场景创建向导
                     ↓
              Step 1: 灾难类型选择
              Step 2: 世界规则设定
              Step 3: 叙事风格选择
              Step 4: AI补充生成完整种子
```

## 游戏交互流程

### 单回合流程

```
玩家输入/选择
    ↓
AI 解析意图
    ↓
状态更新 (属性、背包、状态)
    ↓
事件判定 (随机事件/关键事件)
    ↓
    ├─── 普通回合 → 生成响应 + 下一步选项
    └─── 关键事件 → 触发后台小说章节生成
    ↓
返回响应 (文字描述 + 属性变化 + 选项)
```

### 关键事件判定标准

| 触发条件 | 示例 |
|---------|------|
| 首次遇到重要NPC | 发现其他幸存者 |
| 重大环境变化 | 暴风雪来袭、避难所被毁 |
| 属性临界点 | 生命低于20%、精神崩溃 |
| 重要物品获取 | 获得疫苗、地图 |
| 玩家主动请求 | 输入"记录这段经历" |

## 小说生成系统

### 生成流程

```
游戏开始 → 生成小说骨架 (3-5章结构)
    ↓
游戏进行 → 遇到关键事件 → 触发章节生成
    ↓                      ↓
    └──────────────────────┤
                           1. 读取骨架
                           2. 读取上一章
                           3. 读取当前事件
                           4. 调用新AI实例
                           5. 生成章节内容
    ↓
游戏结束 → 生成结局章节 → 整合输出完整小说
```

### 文学化处理原则

**关键原则**：游戏属性服务于玩家决策，小说内容服务于读者体验

| 游戏数据 | 小说转化方向 |
|---------|-------------|
| 生命值下降 | 痛觉、虚弱感、濒死体验 |
| 精神值下降 | 幻觉、恐惧、记忆混乱 |
| 饥饿/口渴 | 生理痛苦、对食物的渴望 |
| 获得物品 | 情感意义、希望/安全感 |
| 遭遇NPC | 对话、心理活动、关系变化 |

### AI Prompt框架

```
你是一位末世小说作家。根据以下信息撰写章节：

【场景背景】全球冰封，文明崩溃
【当前情节】玩家刚从避难所苏醒
【角色状态】虚弱、愤怒、警觉
【关联角色】可疑的脚印

写作要求：
1. 纯文学化，绝不要出现数值描述
2. 重点描写：感官体验、内心活动、环境氛围
3. 用"展示而非讲述"的手法
4. 保持与前文的连贯性
5. 字数：1000-1500字
```

## 数据结构

### 核心模型

```python
class ScenarioSeed(BaseModel):
    id: str
    name: str
    premise: str
    world_rules: dict
    survival_mechanics: dict
    possible_events: list[str]
    narrative_style: str
    is_custom: bool = False

class PlayerState(BaseModel):
    # 生存属性
    health: int = 100
    hunger: int = 100
    thirst: int = 100
    temperature: int = 37
    sanity: int = 100
    # 能力属性
    strength: int = 50
    agility: int = 50
    intelligence: int = 50
    charisma: int = 50
    perception: int = 50
    # 状态标签
    conditions: list[str] = []
    # 背包
    inventory: list[Item] = []
    max_capacity: int = 20

class GameSession(BaseModel):
    session_id: str
    scenario_id: str
    player_state: PlayerState
    current_location: str
    time_elapsed: int
    history: list[GameTurn] = []
    novel_id: str | None = None
    status: str  # active, ended

class NovelProject(BaseModel):
    novel_id: str
    session_id: str
    skeleton: NovelSkeleton
    chapters: list[NovelChapter] = []
```
